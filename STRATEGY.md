## 1. 市場分析

### 日本の資格市場規模

| 試験 | 年間受験者数 | 備考 |
|------|------------|------|
| TOEIC L&R | 約 200万人 | 国内最大規模の語学試験 |
| 日商簿記 | 約 50万人 | 3級は就活定番 |
| 基本情報技術者試験 | 約 20万人 | IT系登竜門 |
| 実用英語技能検定（英検） | 約 400万人 | 小中高含む |
| 宅地建物取引士 | 約 23万人 | 不動産必須資格 |
| FP技能士 | 約 40万人 | 生涯学習需要 |

**→ ターゲット可能な年間受験者数の合計: 推定 1,000万人超（重複あり）**

### 市場トレンド

- **リスキリング需要の急増**: 経済産業省・文科省が資格取得支援を強化中
- **社会人学習市場の拡大**: EdTech市場は2030年までに年率15%成長予測
- **スマホ学習の定着**: 20〜40代の学習者の70%以上がスマホで学習情報収集
- **副業・転職ブーム**: 資格取得で市場価値を高めたい層が増加

## 2. ユーザーペルソナ

### ペルソナA「IT転職志望の20代」（最重要）
- 年齢: 22〜29歳
- 状況: 未経験からIT業界へ転職検討
- 受験資格: 基本情報、応用情報、G検定、AWS等クラウド系
- 課題: 複数の試験日程が管理できない、試験料がいくらかかるか把握できない
- 支払い意欲: 月500〜1,000円なら払える
- SNS利用: Twitter/X、YouTube

### ペルソナB「TOEIC900点を目指す会社員」
- 年齢: 25〜35歳
- 状況: 昇進・海外赴任に向けてTOEICスコアアップ中
- 受験資格: TOEIC、英検、IELTS
- 課題: スコアの推移を記録したい、次の申込期限を忘れがち
- 支払い意欲: 月300〜800円
- SNS利用: Instagram、Twitter/X

### ペルソナC「資格マニアの30〜40代」
- 年齢: 30〜45歳
- 状況: 20〜30種類の資格保有・挑戦中
- 受験資格: 多岐にわたる
- 課題: 保有資格の一覧管理、更新期限の把握
- 支払い意欲: 年額3,000〜5,000円
- SNS利用: Twitter/X、Note

### ペルソナD「就活中の大学生」
- 年齢: 20〜22歳
- 状況: 就活でTOEIC・日商簿記・ITパスポートを取得中
- 課題: 無料で使いたいが、機能が欲しい
- 支払い意欲: 低（学割があれば払う可能性あり）
- SNS利用: Twitter/X、TikTok

## 4. マネタイズ戦略の選択肢と推奨案

### 選択肢の比較

| モデル | 初期収益 | 継続収益 | ユーザー獲得難易度 | 推奨度 |
|--------|---------|---------|-----------------|--------|
| 完全無料（広告） | △ | △ | ◎ | ▲ |
| 買い切り（¥500〜980） | ○ | ✕ | ○ | ▲ |
| 月額サブスク | △ | ◎ | ○ | ○ |
| **年額サブスク** | ○ | ◎ | ○ | **◎** |
| フリーミアム + 年額 | △→◎ | ◎ | ◎ | **◎◎** |
| 有償アプリ（App Store） | ○ | ✕ | △ | ▲ |

### 推奨モデル: **フリーミアム × 年額サブスクリプション**

#### 理由

1. **無料で始められる** → ユーザー獲得のハードルが最低
2. **価値を体験してから課金** → 離脱率が低い、クレームが少ない
3. **年額 > 月額** → 解約率が低い（月額は3ヶ月で半数が解約するが年額は年間継続率70%超）
4. **買い切りは収益天井が低い** → ユーザー数が停滞すると収入がゼロに近づく
5. **広告はUX破壊** → 学習アプリで広告は最もユーザーに嫌われる要素

## 5. デプロイ戦略

### ステージ別推奨構成

#### フェーズ1: MVP公開（今すぐ）— コスト月額0〜3,000円

```
フロントエンド: Vercel（無料プラン）
バックエンド: Railway（無料枠 $5/月〜）
DB: SQLite on Railway（または PlanetScale 無料枠）
ドメイン: お名前.com / ムームードメイン（¥1,000〜/年）
```

**デメリット**: Railwayの無料枠は月500時間制限あり → 有料化後はすぐ $5/月プランへ

#### フェーズ2: ユーザー100人超（月額3,000〜10,000円）

```
フロントエンド: Vercel Pro（$20/月）またはそのまま無料
バックエンド: Railway $5/月プラン
DB: SQLiteからPostgreSQLへ移行検討（Supabase or Neon）
決済: Stripe 導入（手数料3.6%）
メール: Resend.com（無料枠3,000通/月）
```

#### フェーズ3: 有料会員100人超（月額10,000〜50,000円）

```
インフラ: さくらVPS or AWS Lightsail（¥2,000〜5,000/月）
DB: PostgreSQL（SQLiteは複数アクセスに弱いため移行必須）
CDN: Cloudflare（無料）
監視: Uptime Robot（無料）
```

### 重要: SQLite → PostgreSQL 移行タイミング

現在のSQLiteはOneDrive上で使用しており、本番環境には不向きです。
**有料ユーザーが出る前に必ずPostgreSQLへ移行してください。**

推奨移行先: **Neon.tech**（PostgreSQL、無料枠あり、接続プール対応）

## 6. モバイル展開戦略（iOS / Android）

### 選択肢

#### Option A: PWA（Progressive Web App）★推奨 フェーズ1

**費用**: ほぼ0円  
**工数**: 1〜2週間  
**概要**: Webアプリをスマホのホーム画面に追加できるようにする

```
対応内容:
- manifest.json の設定
- Service Worker でオフライン対応
- アイコン・スプラッシュ画像の設定
- iOS: ホーム画面追加でほぼアプリ感覚
- Android: Chrome が「インストール」プロンプトを表示
```

**メリット**: App Store審査不要、今すぐ配布可能  
**デメリット**: iOSはプッシュ通知に制限あり（iOS16.4以降は対応）

---

#### Option B: Capacitor（既存Reactコードをネイティブ化）★推奨 フェーズ2

**費用**: Apple Developer $99/年（約¥14,000）+ Google Play $25（¥4,000 一回）  
**工数**: 2〜4週間  
**概要**: 既存のReact/TypeScriptコードをそのままネイティブアプリに変換

```
技術スタック:
- Capacitor.js（Ionic製、React対応）
- iOS: Xcode でビルド → App Store 提出
- Android: Android Studio でビルド → Google Play 提出
```

**メリット**: コード再利用率90%以上、プッシュ通知・課金API利用可能  
**デメリット**: ネイティブより若干パフォーマンス劣る

---

#### Option C: React Native（フルネイティブ）フェーズ3以降

**費用**: 上記 + 大幅な開発工数  
**工数**: 3〜6ヶ月（ほぼフルリライト）  
**概要**: iOS/Android ネイティブアプリとしてゼロから構築

**メリット**: 最高のパフォーマンス、ネイティブ機能フル活用  
**デメリット**: 現在のWebコードはほぼ使えない

---

### モバイル推奨ロードマップ

```
今すぐ → フェーズ1: PWA対応（2週間）
3ヶ月後 → フェーズ2: Capacitor でiOS/Android化
           App Store / Google Play に提出
6ヶ月後 → フェーズ3: プッシュ通知、App内課金対応
1年後   → 必要であればReact Native移行検討
```

## 7. フリーミアム設計（機能の線引き）

### 無料プラン（Free）

| 機能 | 制限 |
|------|------|
| 資格一覧の閲覧 | 全件閲覧可能 |
| 資格の保有登録 | **最大5件まで** |
| スコア記録 | ✕（プレミアムのみ） |
| 取得日記録 | ✕（プレミアムのみ） |
| カレンダー表示 | ✕（プレミアムのみ） |
| 試験日リマインダー | ✕ |
| データエクスポート | ✕ |
| 広告表示 | なし（UX優先） |

### プレミアムプラン（Premium）

| 機能 | 内容 |
|------|------|
| 保有資格登録 | 無制限 |
| スコア記録・推移グラフ | ◎ |
| 取得日管理 | ◎ |
| カレンダービュー | ◎ |
| 試験締切リマインダー（メール/プッシュ） | ◎（重要！） |
| データCSVエクスポート | ◎ |
| 複数デバイス同期 | ◎ |
| 優先サポート | ◎ |

### ユーザーを有料転換させる「壁」の設計

> **重要原則**: 無料ユーザーが「もっと使いたい」と思った瞬間に壁を作る

1. **5件登録後**に「プレミアムにアップグレードで無制限に」と表示
2. **カレンダーページ**へのアクセス時に「プレミアム機能です」と表示
3. **試験日30日前**にメールで「リマインダー設定はプレミアムで」と訴求
4. **スコア入力しようとした際**にアップセル

## 8. 価格設定

### 推奨価格

| プラン | 月額換算 | 請求 | 備考 |
|--------|---------|------|------|
| **年額プラン** | **¥300/月** | **¥3,600/年** | メイン訴求 |
| 月額プラン | ¥480/月 | ¥480/月 | 年額との差を見せる |
| 学割プラン | ¥180/月 | ¥2,160/年 | 学生メールアドレス認証 |

### 価格設定の根拠

- **¥3,600/年（¥300/月）**は「Spotifyより安い、コンビニコーヒー3杯分」という訴求が効果的
- 競合のStudyPlus Premiumが¥600/月 → 半額で資格特化機能を提供
- 年額一括払いで**解約率を下げる**（月額の年間継続率40%→年額は70%以上）
- 学割で**大学生層を早期獲得**（卒業後も継続利用する傾向が強い）

### 将来の価格帯（有料会員数増加後）

- チーム/法人プラン: ¥1,980/月（5アカウントまで）→ 企業研修・人事部向け
- 企業向け一括契約: 要見積もり

## 10. マーケティング・集客戦略

## 12. リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| App Store審査リジェクト | 高 | 課金はWebで行いApp内課金は後回し。まずPWA展開 |
| スクレイピングブロック | 中 | 各試験公式情報のキャッシュ保持。管理者手動更新UIを維持 |
| 個人情報漏洩 | 高 | HTTPS必須、パスワードbcrypt、JWTの適切な管理。プライバシーポリシー必須 |
| 競合の模倣 | 中 | ユーザーコミュニティ（SNS）と先行者優位で差別化 |
| 解約率増加 | 中 | エンゲージメント機能（通知、ストリーク）で継続率を高める |
| SQLiteのスケール問題 | 高 | **早期にPostgreSQLへ移行（フェーズ0で実施）** |
