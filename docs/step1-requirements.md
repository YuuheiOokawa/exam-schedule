# Step 1: 要件整理

## プロダクト概要

資格試験スケジュール管理Webアプリ。ユーザーが関心のある資格を登録し、試験日・申込期間・合格発表日をカレンダーで一元管理できる。

---

## ターゲットユーザー

| ユーザー種別 | 特徴 |
|-------------|------|
| 資格取得を目指す社会人 | 複数資格を並行して検討・受験する |
| 学生・就活生 | IT資格や語学資格を計画的に取得したい |
| 資格管理担当者（企業） | 社員の資格取得を管理・推進する |

---

## 機能要件

### MVP（最初に作る）

| # | 機能 | 説明 |
|---|------|------|
| 1 | 資格一覧表示 | 検索・カテゴリ絞り込みで資格を一覧表示 |
| 2 | 資格詳細表示 | 試験日・申込期間・受験料・公式URLを表示 |
| 3 | カレンダー表示 | 試験日・申込期間・合格発表日をカレンダーに表示 |
| 4 | 手動スケジュール登録 | 管理画面から日程を手動入力 |
| 5 | 資格管理（CRUD） | 資格の追加・編集・削除 |
| 6 | SQLite保存 | ローカルDBへの永続化 |

### MVP後に追加

| # | 機能 | 優先度 |
|---|------|--------|
| 7 | 公式サイト自動取得 | 高 |
| 8 | 差分更新 | 中 |
| 9 | 通知機能 | 中 |
| 10 | キャッシュ | 低 |

---

## 資格情報として表示する項目

| フィールド | 必須 | 説明 |
|-----------|------|------|
| 試験日 | - | YYYY-MM-DD または「随時」 |
| 申し込み開始日 | - | YYYY-MM-DD |
| 申し込み締切日 | - | YYYY-MM-DD |
| 合格発表日 | - | YYYY-MM-DD |
| 受験料 | - | テキスト形式 |
| 公式ページURL | ◎ | 必須 |
| 取得元情報 | - | スクレイピング元URL |
| 最終取得日時 | - | 自動記録 |

---

## 非機能要件

| 項目 | 要件 |
|------|------|
| パフォーマンス | 初期表示 2秒以内 |
| 対応ブラウザ | Chrome・Firefox・Safari・Edge 最新版 |
| レスポンシブ | スマホ（375px〜）〜デスクトップ（1440px） |
| ダークモード | 対応必須 |
| アクセシビリティ | WCAG 2.1 AA 準拠を目指す |
| セキュリティ | OWASP Top 10 準拠 |
| 保守性 | 新資格追加がファイル1枚で完結 |

---

## 対応資格スコープ

「ITだけでなくあらゆる資格」を対象とする。

| 大カテゴリ | 小カテゴリ例 |
|-----------|------------|
| 国家資格 | IT・情報、医療・福祉、法律・行政、会計・税務、建設・不動産、金融・証券、工業・電気、運輸・交通、食品・調理、教育、語学 |
| 民間資格 | クラウド（AWS/GCP/Azure）、DB、プログラミング言語、セキュリティ、ネットワーク、PM、データ・AI |
| 公的資格 | 簿記、英検、TOEIC、HSK など |

---

## データ取得方針

| 優先順位 | 方法 | 対象 |
|---------|------|------|
| 1位 | 公式API | IPAなど（存在する場合） |
| 2位 | 公式HTMLスクレイピング | AWS、主要IT資格 |
| 3位 | PDF解析 | 年度別試験要項など |
| 4位 | 手動登録 | 医療・法律等、構造が不定の資格 |

- スクレイピング間隔：3秒以上
- 失敗してもアプリ全体を止めない
- 取得ログを保存する

---

## 制約事項

- ローカル運用想定（初期）
- 外部認証なし（初期）
- 通知機能はMVP後
- スクレイピング対象の利用規約を遵守する
