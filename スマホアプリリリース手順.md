## 3. 方法A：PWAとして公開（最短・推奨）

### 概要
Vercelにデプロイするだけで、ユーザーがホーム画面に追加できます。コード変更は最小限です。

### 手順

#### 3-1. manifest.json を強化

`frontend/public/manifest.json` を以下の内容に更新：

```json
{
  "name": "資格スケジュール",
  "short_name": "資格スケジュール",
  "description": "資格試験の日程・申込期間・合格発表日をカレンダーで一元管理",
  "start_url": "/",
  "scope": "/",
  "display": "standalone",
  "orientation": "portrait",
  "theme_color": "#4f46e5",
  "background_color": "#080f1e",
  "lang": "ja",
  "icons": [
    {
      "src": "/icon-192.png",
      "sizes": "192x192",
      "type": "image/png",
      "purpose": "any"
    },
    {
      "src": "/icon-192-maskable.png",
      "sizes": "192x192",
      "type": "image/png",
      "purpose": "maskable"
    },
    {
      "src": "/icon-512.png",
      "sizes": "512x512",
      "type": "image/png",
      "purpose": "any"
    },
    {
      "src": "/icon-512-maskable.png",
      "sizes": "512x512",
      "type": "image/png",
      "purpose": "maskable"
    }
  ],
  "categories": ["education", "productivity"],
  "shortcuts": [
    {
      "name": "資格一覧",
      "url": "/list",
      "icons": [{ "src": "/icon-192.png", "sizes": "192x192" }]
    },
    {
      "name": "カレンダー",
      "url": "/calendar",
      "icons": [{ "src": "/icon-192.png", "sizes": "192x192" }]
    }
  ]
}
```

> **SVG→PNG変換が必要**（後述の[セクション6](#6-共通アイコンスプラッシュ画像の整備)参照）

#### 3-2. iOS向けメタタグを追加

`frontend/index.html` の `<head>` に追加：

```html
<!-- iOS PWA -->
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
<meta name="apple-mobile-web-app-title" content="資格スケジュール">
<link rel="apple-touch-icon" href="/apple-touch-icon.png">
<link rel="apple-touch-startup-image" href="/splash-2048x2732.png" media="(device-width: 1024px) and (device-height: 1366px)">
<link rel="apple-touch-startup-image" href="/splash-1668x2224.png" media="(device-width: 834px) and (device-height: 1112px)">
<link rel="apple-touch-startup-image" href="/splash-750x1334.png" media="(device-width: 375px) and (device-height: 667px)">
```

#### 3-3. Vercelにデプロイ

```bash
# フロントエンドをビルド
cd frontend
npm run build

# Vercel CLIでデプロイ（初回）
npx vercel --prod
```

#### 3-4. ユーザーへのインストール案内

**Android（Chrome）：**
1. アプリのURLをChromeで開く
2. アドレスバー右の「インストール」アイコンをタップ
3. 「インストール」→ホーム画面に追加される

**iOS（Safari）：**
1. Safariでアプリを開く
2. 下部の「共有」ボタン → 「ホーム画面に追加」
3. 「追加」でホーム画面に追加される

## 4. 方法B：Google Play Store（TWA）

### 概要
TWA（Trusted Web Activity）を使うとWebアプリをそのままGoogle Playで配信できます。
ネイティブコードを書かずにPlay Storeに掲載できます。

### 前提条件
- Vercelでの本番デプロイが完了していること
- Androidの実機またはエミュレータ
- Android Studio インストール済み

### 手順

#### 4-1. Google Play Console 登録

1. [Google Play Console](https://play.google.com/console) にアクセス
2. Googleアカウントでログイン
3. デベロッパー登録料 **$25（一回のみ）** を支払い
4. 「アプリを作成」→ アプリ名・言語・種類を入力

#### 4-2. Bubblewrap CLI でTWAプロジェクト生成

```bash
# Bubblewrapインストール
npm install -g @bubblewrap/cli

# TWAプロジェクト初期化（プロジェクトルートで実行）
mkdir twa && cd twa
bubblewrap init --manifest https://your-domain.vercel.app/manifest.json
```

対話形式で以下を入力：

| 項目 | 入力値 |
|------|-------|
| Application ID | `jp.co.yourname.shikakuschedule` |
| App name | 資格スケジュール |
| Short name | 資格スケジュール |
| Host | `your-domain.vercel.app` |
| Start URL | `/` |
| Theme color | `#4f46e5` |
| Background color | `#080f1e` |

#### 4-3. Digital Asset Links の設定

TWAがドメインを信頼するための設定。フロントエンドに追加：

`frontend/public/.well-known/assetlinks.json` を作成：

```json
[{
  "relation": ["delegate_permission/common.handle_all_urls"],
  "target": {
    "namespace": "android_app",
    "package_name": "jp.co.yourname.shikakuschedule",
    "sha256_cert_fingerprints": ["XXXX:XXXX:...（後述のキーストアから取得）"]
  }
}]
```

#### 4-4. APK/AABをビルド

```bash
cd twa

# デバッグビルド（テスト用）
bubblewrap build

# リリースビルド（署名付き）
bubblewrap build --release
```

Keystoreのパスワードを設定（初回のみ）。

#### 4-5. SHA256フィンガープリントを取得

```bash
keytool -list -v \
  -keystore ./android.keystore \
  -alias android \
  -storepass YOUR_PASSWORD \
  -keypass YOUR_PASSWORD
```

出力の `SHA256:` の値を `assetlinks.json` に貼り付け、Vercelに再デプロイ。

#### 4-6. Play Consoleでアプリ提出

1. Play Console → 「内部テスト」→ AABファイルをアップロード
2. ストア掲載情報を入力：
   - アプリアイコン（512×512 PNG）
   - フィーチャーグラフィック（1024×500 PNG）
   - スクリーンショット（最低2枚）
   - 説明文
   - カテゴリ：「教育」
   - コンテンツレーティング：設定
   - プライバシーポリシーURL
3. 「内部テスト」→「本番」に昇格

**審査期間：** 初回は1〜3日程度

## 5. 方法C：App Store（Capacitor）

### 概要
CapacitorはWebアプリをiOS/Androidのネイティブアプリに変換するツールです。
App Store（iOS）にも掲載できます。

### 前提条件
- **Mac が必要**（iOSビルドはmacOS必須）
- Xcode インストール済み（Mac App Storeから無料）
- Apple Developer Program 登録（**$99/年**）
- Android Studio インストール済み

### 手順

#### 5-1. Capacitorのインストール

```bash
cd frontend

npm install @capacitor/core @capacitor/cli
npm install @capacitor/ios @capacitor/android

npx cap init "資格スケジュール" "jp.co.yourname.shikakuschedule" --web-dir dist
```

#### 5-2. フロントエンドをビルド

```bash
npm run build
```

#### 5-3. iOSプロジェクトを追加

```bash
npx cap add ios
npx cap sync ios
npx cap open ios  # Xcodeが開く
```

#### 5-4. Xcodeで設定

1. `App` ターゲットを選択
2. **Signing & Capabilities** タブ
   - Team：Apple Developerアカウントを選択
   - Bundle Identifier：`jp.co.yourname.shikakuschedule`
3. **General** タブ
   - Display Name：資格スケジュール
   - Version：1.0.0
4. アイコン設定：`Assets.xcassets` → `AppIcon` に各サイズのPNGを追加

#### 5-5. Androidプロジェクトを追加

```bash
npx cap add android
npx cap sync android
npx cap open android  # Android Studioが開く
```

#### 5-6. App Store Connect でアプリ登録

1. [App Store Connect](https://appstoreconnect.apple.com) にログイン
2. 「新しいApp」→ 各情報を入力
   - プラットフォーム：iOS
   - 名前：資格スケジュール
   - バンドルID：`jp.co.yourname.shikakuschedule`
   - SKU：任意の文字列
3. Xcodeで **Product → Archive** → App Store Connectにアップロード
4. スクリーンショット・説明文・プライバシーポリシーを追加
5. 「審査に提出」

**審査期間：** 初回は1〜7日程度

## 6. 共通：アイコン・スプラッシュ画像の整備

現在のアイコンはSVG形式です。ストア申請にはPNG形式が必要です。

### 必要なアイコンサイズ

| 用途 | サイズ | 形式 |
|------|--------|------|
| PWA・TWA 標準 | 192×192 | PNG |
| PWA・TWA 標準 | 512×512 | PNG |
| PWA maskable（Androidホーム画面用） | 192×192 | PNG |
| Play Store掲載 | 512×512 | PNG |
| App Store掲載 | 1024×1024 | PNG（角丸なし） |
| iOS ホーム画面 | 180×180 | PNG |
| iOS スプラッシュ | 各解像度 | PNG |

### 変換方法（オンラインツール）

1. **[RealFaviconGenerator](https://realfavicongenerator.net/)**
   - SVGをアップロードして全サイズ一括生成

2. **[PWA Asset Generator](https://github.com/elegantapp/pwa-asset-generator)**
   ```bash
   npm install -g pwa-asset-generator
   pwa-asset-generator frontend/public/icon-512.svg frontend/public \
     --index frontend/index.html \
     --manifest frontend/public/manifest.json \
     --splash-only --portrait-only
   ```

### アイコンデザインのガイドライン

- 背景色：`#4f46e5`（ブランドカラー）
- Maskable アイコン：中央の「資」文字が Safe Zone（中央72%以内）に収まること
- App Store：完全な正方形PNG、透過なし、角丸なし（ストアが自動で丸める）

- [ ] Chromeで「インストール」プロンプトが表示される
